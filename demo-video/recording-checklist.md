# Recording Checklist

## Before recording

- Confirm Docker Desktop is running.
- Confirm the Compose stack is healthy.
- Close email, chat, and personal applications.
- Enable Windows Do Not Disturb.
- Hide desktop files and browser bookmarks.
- Use a 1920×1080 recording canvas.
- Increase terminal font size.
- Use a clean PowerShell terminal.
- Verify microphone input and volume.
- Verify the GitHub repository is public.
- Run a clean Docker Compose startup.
- Run `./scripts/video-demo.ps1 -NoPause`.
- Confirm all eight mandatory scenarios pass.
- Confirm Git status is clean.
- Confirm no tokens, keys, or authorization headers appear.
- Open all HTML cards before recording.
- Open the architecture visual in a readable rendered form.
- Disable automatic terminal transparency if it harms readability.

## During recording

- Start with the title card.
- Show the architecture diagram.
- Run the real live demo.
- Avoid unnecessary scrolling.
- Speak slowly and clearly.
- Do not silently restart failed commands.
- Show the security summary.
- Explain the GPT-5.6 and Codex contribution.
- Show the public GitHub repository.
- End with the closing card.
- Keep the complete video under three minutes.

## After recording

- Verify the duration is below three minutes.
- Verify narration is clearly audible.
- Verify terminal and diagram text are readable at 1080p.
- Check that no personal notifications appeared.
- Check that no secrets or tokens are visible.
- Trim only dead time; do not remove evidence of failures.
- Add the supplied subtitles.
- Upload to YouTube as public or unlisted according to the hackathon rules.
- Verify the YouTube link in a private browser.
- Add the final video URL to Devpost.
- Watch the uploaded YouTube version once before submission.
